# 클린 코드 계획서

> 목적: 구조(아키텍처) 변경 없이 **가독성·일관성·중복 제거** 중심의 클린 코드를 진행한다.
> 아키텍처를 바꾸는 **리팩토링은 별도**로 사용자가 방향·구조를 지정한다 (이 문서 범위 밖).

## 진행 방식
1. 아래 항목을 **하나씩** 진행한다.
2. 각 항목 완료 후 사용자가 확인하고 **confirm** 하면 그 항목은 종료(✅) 처리한다.
3. confirm 전에는 다음 항목으로 넘어가지 않는다.

## 범위 기준
- **포함(클린 코드)**: 임포트 정리, 죽은 코드/중복 리터럴 제거, 매직 값→상수, 포맷·정렬 일관성, 작은 헬퍼 추출, 주석·문서 최신화.
- **제외(리팩토링)**: 파일 분리, 모듈/라우터 재구성, 데이터 흐름 변경 등 → 문서 하단 "리팩토링 후보" 참고.

## 항목 상세

### CC-01 · 인라인 import 상단 이동
- **대상**: `main.py` (L126 `text`, L227 `re, requests`, L279 `requests`, L301 `yfinance`), `collector.py` (L182 `func, and_`, L208 `desc`, L492 `text`, L509/565 `or_`, L566 `GroupMember, UserGroup`)
- **문제**: 함수 내부에 import가 흩어져 있음. 특히 `collector.collect_all`은 **for 루프 안에서** `from sqlalchemy import func as _func`를 매 반복 실행.
- **개선**: 표준 라이브러리/서드파티/내부 모듈 순서로 파일 상단에 모은다. (단, 무거운 선택적 의존성은 유지 여부 검토)
- **위험**: 낮음 (동작 동일).

### CC-02 · 불필요한 별칭 제거
- **대상**: `collector.py` `fill_metrics` L510-512
- **문제**: `tickers = [...]` 후 `stocks = tickers`로 재할당하고 `for ticker in stocks`. 별칭이 의미 없음.
- **개선**: `stocks` 별칭 삭제, `tickers` 직접 사용. 반환부 `len(stocks)`→`len(tickers)`.
- **위험**: 낮음.

### CC-03 · 오해 유발 주석 정리
- **대상**: `templates/chart.html` `toggleGroupAccordion` (L1326 부근)
- **문제**: 주석은 "shift 클릭이면 전체 선택/해제"라고 적혀 있으나 실제 코드는 shift 처리 없이 아코디언 토글만 함.
- **개선**: 주석을 실제 동작에 맞게 수정(또는 함수가 단순 위임뿐이면 정리).
- **위험**: 낮음.

### CC-04 · 기간 화이트리스트 통합
- **대상**: `main.py` `backfill`(L150), `fetch_group_data`(L432)의 `{"6mo","1y","2y","5y","max"}` + `_PERIOD_DAYS`(L163)
- **문제**: 같은 기간 집합이 두 곳에 하드코딩, 일 수 매핑은 또 다른 dict로 분리.
- **개선**: 기간 정의를 한 곳(모듈 상수)으로 모으고 검증·일수 매핑을 그 단일 출처에서 파생.
- **위험**: 낮음.

### CC-05 · "max" 매직넘버 의미 부여
- **대상**: `main.py` `_PERIOD_DAYS`의 `"max": 99999`
- **문제**: 99999가 "사실상 무제한"을 의미하나 설명 없음.
- **개선**: 명명 상수 또는 주석으로 의도 표기. (CC-04와 함께 처리 가능)
- **위험**: 낮음.

### CC-06 · 티어 문자열 상수화
- **대상**: `main.py`, `collector.py` 전반의 `"1차"/"2차"/"3차"/"그룹"`
- **문제**: 문자열 리터럴이 여러 곳에 반복. 오타·불일치 위험.
- **개선**: 상수(예: `TIER_GROUP = "그룹"`, 티어 목록)로 정의해 참조. 프런트(JS)와의 값 일치는 유지.

### CC-07 · yfinance 필드명 후보 리스트 상수화

## 리팩토링 후보 (이 문서 범위 밖 · 사용자 지시 대기)
아래는 클린 코드가 아니라 **구조 변경**이라 별도 진행한다.
- `templates/chart.html`의 인라인 `<script>`(약 1,000줄)를 `static/chart.js`로 분리.
- 5개 템플릿에 복붙된 공통 헤더/네비/토스트 CSS를 공유 스타일시트로 추출.
- 스냅샷 생성 로직(`collect_all`/`backfill_history`/`backfill_ticker` 3중복)을 단일 빌더로 통합.
- 라우트를 `APIRouter`로 도메인별(대시보드/그룹/수집) 모듈 분리.
- DB 마이그레이션 방식(`ALTER TABLE` try/except 루프) 개선.
- 외부 시세/검색 호출(yfinance·네이버)을 서비스 계층으로 분리.
